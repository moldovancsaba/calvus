
# Technical Architecture

## Why MongoDB via Supabase Edge?
- Chosen for user request, mesh’s naturally hierarchical structure, and flexible future expansion.
- Hosted on MongoDB Atlas, accessed securely via Supabase Edge.

## Why 5s Polling vs WebSockets?
- App uses simple, reliable 5s polling (no persistent socket) for all activity.
- This balances low server load, compatibility, and user expectation for “live enough” updates.

## Geometry & Subdivision

- The mesh’s **18 base triangles** are generated by a canonical geodesic mesh formula (see code in geometry.ts).
- Subdivision of each triangle occurs on 11th tap, up to level 19. All subdivisions maintain geometric and index integrity.

## Data Model (2025+)
- Triangles use canonical “dot notation” IDs (e.g., `1.3.2` for hierarchies/subdivisions).
- **State is stored/loaded exclusively via the latest per-triangle “activity snapshot”.**
- Rebuilding the mesh simply applies the last known state for each triangle, with no incremental log replay.
